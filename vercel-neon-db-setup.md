# Налаштування підключення Laravel до бази даних Neon PostgreSQL на Vercel

Ця інструкція крок за кроком описує процес налаштування підключення вашого локального проєкту Laravel до віддаленої бази даних Neon PostgreSQL, використовуючи змінні середовища, отримані з Vercel.

## Крок 1: Отримання конфігурації з Vercel

Спочатку завантажте ваші змінні середовища для проєкту локально за допомогою команди Vercel CLI (запустіть у терміналі в корені проєкту):

```bash
vercel env pull .env.local
```

Це створить файл `.env.local`, який міститиме всі необхідні доступи.

## Крок 2: Оновлення файлу `.env`

Відкрийте файл `.env.local` та скопіюйте дані бази даних (шукайте ті, що без `_POOLER`, тобто непульовані з'єднання, наприклад `POSTGRES_URL_NON_POOLING`, інакше міграції можуть не працювати).

Відкрийте ваш основний файл `.env` та замініть або оновіть блок конфігурації підключення до бази даних наступним чином:

```env
DB_CONNECTION=pgsql
DB_HOST=ep-restless-pond-ai2uylb3.c-4.us-east-1.aws.neon.tech  # Використовуйте НЕПУЛЬОВАНЕ з'єднання (без pooler в назві)
DB_PORT=5432
DB_DATABASE=neondb
DB_USERNAME=neondb_owner
DB_PASSWORD=npg_Yan1GuSztTP8   # Ваш пароль з .env.local
DB_SSLMODE=require
```

> **Важливо:** Для міграцій (структурних змін БД) необхідно використовувати `DB_HOST` **без** слова `pooler`. З пульованим з'єднанням (PgBouncer) міграції впадуть з помилкою відхилення транзакції.

## Крок 3: Увімкнення PHP розширення PDO PostgreSQL

Щоб PHP міг працювати з базою даних PostgreSQL, необхідно переконатися, що відповідне розширення увімкнене у вашому `php.ini`.

1. Знайдіть, який саме файл `php.ini` зараз використовує ваш PHP для командного рядка:
    ```bash
    php --ini
    ```
2. Відкрийте знайдений файл `php.ini` у текстовому редакторі.
3. Знайдіть наступні рядки (вони можуть мати крапку з комою `;` на початку) і розкоментуйте їх (видаліть `;`):
    ```ini
    extension=pdo_pgsql
    extension=pgsql
    ```
4. Збережіть файл.

## Крок 4: Очищення кешу конфігурації Laravel

Щоб Laravel підхопив ваші нові налаштування з `.env`, очистіть кеш:

```bash
php artisan config:clear
```

## Крок 5: Запуск міграцій

Тепер ви можете створити таблиці у вашій віддаленій базі даних Neon. Запустіть міграції:

```bash
php artisan migrate
```

Якщо це абсолютно нова база даних або під час попередніх спроб виникали конфлікти (наприклад, помилки типу _"current transaction is aborted"_), ви можете повністю очистити її та перестворити таблиці цією командою:

```bash
php artisan migrate:fresh
```

---

**Готово!** Ваша база даних на Neon підключена, а всі таблиці створені та готові до роботи локально та на продакшені у Vercel.
